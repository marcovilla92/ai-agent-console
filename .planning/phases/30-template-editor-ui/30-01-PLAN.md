Add a new endpoint `GET /templates/{template_id}/files` that:
1. Validates template exists in registry (reuse `_get_entry_or_404`)
2. Walks `TEMPLATES_ROOT / template_id` using `rglob("*")`, skipping directories and EXCLUDE_DIRS
3. For each file, reads content as UTF-8 text (skip binary files by catching UnicodeDecodeError -- set content to "[binary file]" for those)
4. Returns `TemplateFilesResponse(files={rel_path: content})`

Place the route BEFORE the `/{template_id}` route to avoid path conflicts (or use a distinct path). Actually, since FastAPI matches routes in order and `/files` is a fixed segment after the template_id, add it as:
`@template_router.get("/{template_id}/files", response_model=TemplateFilesResponse)`

This must be placed BEFORE the generate endpoint registration to avoid path conflicts. Place it right after the `get_template` endpoint.

Add method `loadTemplateFiles(templateId)`:
```javascript
async loadTemplateFiles(templateId) {
  this.templateFiles = {};
  this.templateSelectedFile = null;
  try {
    var resp = await apiFetch('/templates/' + templateId + '/files');
    if (resp.ok) {
      var data = await resp.json();
      this.templateFiles = data.files || {};
    }
  } catch(e) { /* silent */ }
}
```

Add computed getter `templateFileTree` that converts the flat `templateFiles` dict keys into a nested tree structure:
- Parse each path like `src/main.py`, `.claude/agents/planner.md`
- Build a tree of `{name, path, is_dir, children: [], expanded}` nodes
- Sort: directories first, then files, alphabetically within each group
- Use `templateExpandedDirs` to track open/closed state

**Modify `viewTemplateDetail(tmpl)`** to also call `this.loadTemplateFiles(tmpl.id)` after setting selectedTemplate.
