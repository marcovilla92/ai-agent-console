# Phase 29: AI Template Generation - Research

**Researched:** 2026-03-14
**Domain:** AI-driven template generation via Claude CLI structured output, with server-side validation and separate concurrency control
**Confidence:** HIGH

## Summary

Phase 29 adds a single new endpoint (`POST /templates/generate`) that accepts a natural language project description and returns a complete template structure (files, agents, commands, settings, CLAUDE.md) generated by Claude CLI. The implementation reuses the existing `call_orchestrator_claude()` pattern from `src/runner/runner.py` with a different JSON schema tailored for template output. Generated agent and command files are validated through the existing `discover_project_agents()` and `discover_project_commands()` loaders before the response is returned to the client, surfacing any `validation_errors`.

The critical architectural decision is using a **separate `asyncio.Semaphore(1)`** for generation, independent from the pipeline's `Semaphore(2)` in `TaskManager`. This prevents AI generation from consuming pipeline task slots (the VPS has only 7.6GB RAM). When the generation semaphore is busy, the endpoint returns HTTP 429 with a `Retry-After` header.

**Primary recommendation:** Add `POST /templates/generate` to `src/server/routers/templates.py` using `call_orchestrator_claude()` with a generation-specific JSON schema and system prompt. Validate generated files by writing them to a temp directory and running loaders against it. Use a module-level `asyncio.Semaphore(1)` for concurrency control.

### Pattern 1: Structured Output via Claude CLI
**What:** Use `call_orchestrator_claude()` with a JSON schema that describes the full template structure. Claude CLI enforces the schema via `--json-schema` and returns conforming JSON.
**When to use:** When you need a single, complete structured response (not streaming).
**Example:**
```python
# The schema defines what Claude must return
TEMPLATE_GEN_SCHEMA = json.dumps({
    "type": "object",
    "properties": {
        "id": {"type": "string", "pattern": "^[a-z0-9-]+$"},
        "name": {"type": "string"},
        "description": {"type": "string"},
        "files": {
            "type": "object",
            "additionalProperties": {"type": "string"},
            "description": "Map of relative file path to file content"
        }
    },
    "required": ["id", "name", "description", "files"]
})

# Call Claude CLI with structured output
raw = await call_orchestrator_claude(
    prompt=user_description,
    schema=TEMPLATE_GEN_SCHEMA,
    system_prompt=system_prompt_text,
)

# Parse the structured response
response = json.loads(raw)
data = response.get("structured_output") or response.get("result")
if isinstance(data, str):
    data = json.loads(data)
```

### Pattern 2: Non-blocking Semaphore with HTTP 429
**What:** Use `Semaphore.acquire_nowait()` (raises without blocking) to reject requests when generation is already in progress.
**When to use:** For resource-constrained endpoints that should not queue.
**Example:**
```python
_generation_semaphore = asyncio.Semaphore(1)

@template_router.post("/generate")
async def generate_template(req: GenerateRequest):
    try:
        _generation_semaphore.acquire_nowait()  # raises if busy -- note: this is a method, not async
    except ValueError:
        raise HTTPException(
            status_code=429,
            detail="Template generation already in progress",
            headers={"Retry-After": "30"},
        )
    try:
        # ... do generation ...
        return result
    finally:
        _generation_semaphore.release()
```

**Important correction:** `asyncio.Semaphore.acquire_nowait()` does not exist as a standalone method. The correct approach is to check the semaphore's internal counter or use a lock pattern. The practical implementation uses `asyncio.Lock` (which is a Semaphore(1)) or a simple boolean flag guarded by a lock. Alternatively, wrap with `asyncio.wait_for(sem.acquire(), timeout=0)` and catch `TimeoutError`.

### Pattern 3: Temp Directory Validation
**What:** Write generated files to a temp directory, run existing loaders against it, collect validation errors, then clean up.
**When to use:** When you need to validate generated content using existing filesystem-based loaders.
**Example:**
```python
import tempfile
import shutil

async def validate_generated_files(files: dict[str, str]) -> list[str]:
    errors = []
    tmp_dir = tempfile.mkdtemp(prefix="tmpl-gen-")
    try:
        # Write files to temp dir
        for rel_path, content in files.items():
            target = Path(tmp_dir) / rel_path
            target.parent.mkdir(parents=True, exist_ok=True)
            target.write_text(content, encoding="utf-8")

        # Validate agents through loader
        from src.agents.loader import discover_project_agents
        agents = discover_project_agents(tmp_dir)
        # Any .claude/agents/*.md file that didn't produce an AgentConfig is invalid
        agent_dir = Path(tmp_dir) / ".claude/agents"
        if agent_dir.is_dir():
            for md in agent_dir.glob("*.md"):
                name = md.stem.lower().replace(" ", "-")
                # Check reserved names
                if name in ("plan", "execute", "test", "review"):
                    errors.append(f"Agent '{name}' uses reserved core agent name")

        # Validate commands through loader
        from src.commands.loader import discover_project_commands
        commands = discover_project_commands(tmp_dir)
    finally:
        shutil.rmtree(tmp_dir, ignore_errors=True)
    return errors
```

### Anti-Patterns to Avoid
- **Reusing pipeline semaphore for generation:** Generation and pipeline tasks compete for the same slots. The VPS has limited RAM; a generation call while 2 tasks run could OOM.
- **Skipping validation before response:** LLMs produce plausible but structurally invalid output. Returning unvalidated templates means broken projects.
- **Streaming generation output:** Template generation needs the complete JSON response to validate. Streaming partial JSON is not useful and complicates validation.
- **Building a complex prompt template engine:** A single system prompt text file with clear instructions is sufficient. No Jinja2 templating for prompts.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| JSON structured output from LLM | Custom JSON extraction/parsing | `call_orchestrator_claude()` with `--json-schema` | Claude CLI enforces schema; handles retries and edge cases |
| Agent file validation | Custom YAML frontmatter parser | `discover_project_agents()` from `src/agents/loader.py` | Already handles frontmatter, defaults, encoding edge cases |
| Command file validation | Custom command parser | `discover_project_commands()` from `src/commands/loader.py` | Already handles name sanitization and empty files |
| Template file writing | Custom file writer | `safe_write_template_file()` from templates router | Already handles path traversal prevention |
| Reserved agent name checking | Hardcoded name list | `PROTECTED_AGENTS` frozenset from `src/agents/config.py` | Single source of truth for protected names |

**Key insight:** The entire validation pipeline already exists in Phase 26-28 code. The only new logic is the system prompt, the JSON schema, and the endpoint wiring.

## Common Pitfalls

### Pitfall 1: Claude CLI structured_output Parsing
**What goes wrong:** The Claude CLI `--output-format json` response wraps the actual data in a `structured_output` or `result` field, and sometimes double-serializes it as a string.
**Why it happens:** Claude CLI response format varies between versions.
**How to avoid:** Follow the exact parsing pattern from `get_orchestrator_decision()` in `orchestrator.py` lines 257-274: try `structured_output`, then `result`, then try `json.loads()` if the value is a string.
**Warning signs:** Getting the raw wrapper object instead of the template data.

### Pitfall 2: Generated Agent Names Colliding with Core Agents
**What goes wrong:** AI generates an agent named `plan.md` or `review.md`, which would collide with protected core agents.
**Why it happens:** The LLM doesn't know about reserved names unless told.
**How to avoid:** (1) System prompt explicitly lists reserved names. (2) Post-generation validation checks `PROTECTED_AGENTS` frozenset. (3) Include collision as `validation_errors` in response.
**Warning signs:** `merge_registries()` silently skipping project agents with warnings.

### Pitfall 3: Semaphore Not Released on Error
**What goes wrong:** If generation raises an exception after acquiring the semaphore, subsequent requests are permanently blocked with 429.
**Why it happens:** Missing `finally` block for semaphore release.
**How to avoid:** Always use `try/finally` pattern for semaphore release. Or use `async with` if using a Lock.
**Warning signs:** Generation endpoint returns 429 even when nothing is running.

### Pitfall 4: Oversized Generation Prompt
**What goes wrong:** The system prompt + user description exceeds context window, or generates a response too large for the 10MB line buffer in the runner.
**Why it happens:** No size limits on user input or expected output.
**How to avoid:** (1) Cap user description to 2000 chars. (2) System prompt should be concise (~1000 tokens). (3) The 10MB buffer in `stream_claude` already protects the subprocess reader; `call_orchestrator_claude` uses `proc.communicate()` which buffers in memory -- should be fine for template-sized output.
**Warning signs:** Claude CLI subprocess timeout or OOM.

### Pitfall 5: File Path Injection in Generated Templates
**What goes wrong:** AI generates file paths like `../../etc/passwd` or absolute paths.
**Why it happens:** LLM output is untrusted.
**How to avoid:** (1) Validate all generated file paths using `safe_write_template_file()` which checks `is_relative_to()`. (2) Reject paths starting with `/` or containing `..`. (3) Do this before writing to temp dir too.
**Warning signs:** `ValueError: Path traversal attempt` from `safe_write_template_file`.

### Pitfall 6: Empty or Malformed JSON from Claude CLI
**What goes wrong:** Claude CLI returns empty stdout, non-JSON, or schema-violating JSON.
**Why it happens:** API errors, rate limits, malformed prompts.
**How to avoid:** (1) `call_orchestrator_claude` already raises `CalledProcessError` on non-zero exit. (2) Wrap JSON parsing in try/except. (3) Return a clear error response to the client.
**Warning signs:** 500 errors on the generate endpoint.

## Code Examples

### Complete Endpoint Structure
```python
# In src/server/routers/templates.py

import asyncio
import json
import tempfile
import shutil
from pathlib import Path

from src.runner.runner import call_orchestrator_claude
from src.agents.loader import discover_project_agents
from src.agents.config import PROTECTED_AGENTS
from src.commands.loader import discover_project_commands

# Separate semaphore for AI generation (AIGEN-03)
_gen_lock = asyncio.Lock()

TEMPLATE_GEN_SCHEMA = json.dumps({
    "type": "object",
    "properties": {
        "id": {"type": "string"},
        "name": {"type": "string"},
        "description": {"type": "string"},
        "files": {
            "type": "object",
            "additionalProperties": {"type": "string"}
        }
    },
    "required": ["id", "name", "description", "files"]
})


class GenerateTemplateRequest(BaseModel):
    description: str  # Natural language project description
    # Optional hints
    stack: str | None = None


class GenerateTemplateResponse(BaseModel):
    id: str
    name: str
    description: str
    files: dict[str, str]
    validation_errors: list[str]


@template_router.post("/generate", response_model=GenerateTemplateResponse)
async def generate_template(req: GenerateTemplateRequest):
    # AIGEN-03: Non-blocking lock check
    if _gen_lock.locked():
        raise HTTPException(
            status_code=429,
            detail="Template generation already in progress",
            headers={"Retry-After": "30"},
        )

    async with _gen_lock:
        # Build prompt
        prompt = f"Generate a project template for: {req.description}"
        if req.stack:
            prompt += f"\nPreferred stack: {req.stack}"

        # Load system prompt
        system_prompt = _load_gen_system_prompt()

        # Call Claude CLI with structured output
        raw = await call_orchestrator_claude(
            prompt=prompt,
            schema=TEMPLATE_GEN_SCHEMA,
            system_prompt=system_prompt,
        )

        # Parse response (same pattern as orchestrator.py)
        response = json.loads(raw)
        data = response.get("structured_output") or response.get("result")
        if isinstance(data, str):
            data = json.loads(data)

        # AIGEN-02: Validate generated files
        validation_errors = _validate_generated_files(data.get("files", {}))

        return GenerateTemplateResponse(
            id=data["id"],
            name=data["name"],
            description=data["description"],
            files=data.get("files", {}),
            validation_errors=validation_errors,
        )
```

### Validation Function
```python
def _validate_generated_files(files: dict[str, str]) -> list[str]:
    """Validate generated template files through existing loaders."""
    errors = []
    tmp_dir = tempfile.mkdtemp(prefix="tmpl-validate-")
    try:
        for rel_path, content in files.items():
            # Check path safety
            if rel_path.startswith("/") or ".." in rel_path:
                errors.append(f"Invalid path: {rel_path}")
                continue
            target = Path(tmp_dir) / rel_path
            target.parent.mkdir(parents=True, exist_ok=True)
            target.write_text(content, encoding="utf-8")

        # Validate agents
        agents_dir = Path(tmp_dir) / ".claude" / "agents"
        if agents_dir.is_dir():
            for md in agents_dir.glob("*.md"):
                stem = md.stem.lower().replace(" ", "-")
                if stem in PROTECTED_AGENTS:
                    errors.append(f"Agent '{stem}' uses reserved name")
            # Run full loader to catch parse errors
            try:
                discover_project_agents(tmp_dir)
            except Exception as e:
                errors.append(f"Agent validation error: {e}")

        # Validate commands
        try:
            discover_project_commands(tmp_dir)
        except Exception as e:
            errors.append(f"Command validation error: {e}")
    finally:
        shutil.rmtree(tmp_dir, ignore_errors=True)
    return errors
```

## Open Questions

1. **Timeout for Claude CLI generation call**
   - What we know: `call_orchestrator_claude` uses `proc.communicate()` which blocks until completion. No timeout currently.
   - What's unclear: How long template generation typically takes (likely 15-60 seconds).
   - Recommendation: Add `asyncio.wait_for(..., timeout=120)` around the generation call. Return 504 on timeout.

2. **Whether to auto-save the generated template**
   - What we know: The frontend can call `POST /templates` separately to save. The generate endpoint just returns the preview.
   - What's unclear: Whether the user wants a one-step "generate and save" flow.
   - Recommendation: Generate-only endpoint returns preview. Frontend shows the result and lets user save explicitly via existing `POST /templates`. This aligns with Phase 30 (Template Editor) which adds preview-before-save.

3. **JSON schema depth for file content**
   - What we know: Claude CLI `--json-schema` enforces top-level structure but cannot enforce that file contents are valid Python/YAML/etc.
   - What's unclear: How reliably Claude generates valid YAML frontmatter in agent files.
   - Recommendation: The system prompt is the main quality lever. Validation via loaders catches structural issues. Content quality is empirical -- expect iteration on the system prompt.
