Create `src/settings/loader.py` with:
- `CLAUDE_SETTINGS_FILE = ".claude/settings.local.json"`
- `SETTINGS_WHITELIST` -- set of top-level keys project settings can override. Initially: `{"permissions"}`. This is the security boundary -- project settings can only override keys in this whitelist.
- `load_project_settings(project_path: str) -> dict` -- reads `.claude/settings.local.json`, returns parsed dict. Returns `{}` if file missing or invalid JSON (log warning on parse error). Uses `encoding="utf-8"` and `errors="replace"`.
- `merge_settings(global_settings: dict, project_settings: dict) -> dict` -- deep-merges project into global, but ONLY for keys in `SETTINGS_WHITELIST`. Non-whitelisted keys in global are preserved and cannot be overridden. For whitelisted keys, project values win. Returns new dict (no mutation).
