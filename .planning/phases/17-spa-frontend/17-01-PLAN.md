Create `static/index.html` -- the complete single-page application. This is the entire frontend for v2.1.

**Critical locked decisions:**
- x-show (not x-if) for the 4 main view divs -- preserves DOM and WebSocket connections
- Alpine.store('app', {...}) for all shared state -- survives view switches
- Pico CSS via CDN for styling (semantic HTML, no custom classes needed)
- Alpine.js 3 via CDN, no build step

**Structure:**

1. **HTML head:** Same CDN links as existing base.html (Pico CSS 2, Alpine.js 3). Add inline styles for status colors (copy from base.html), log-output max-height, badge styling, and any layout helpers.

2. **Nav bar:** "AI Agent Console" brand. Show selected project name when in prompt/running views.

3. **Four views using x-show:**

   **View: select** (x-show="$store.app.view === 'select'")
   - "Select Project" heading
   - "New Project" button that switches to create view and calls loadTemplates()
   - Project cards: each card shows project.name, stack badges (split stack string by comma, render as small styled spans), timeAgo(last_used_at). Click calls selectProject(project).
   - Empty state: "No projects found" message
   - Load projects on init via loadProjects()

   **View: create** (x-show="$store.app.view === 'create'")
   - "Create Project" heading with back button to select view
   - Form fields: name (text input, required), description (textarea, optional), template (select dropdown populated from $store.app.templates)
   - Submit button calls createProject(name, description, template). On success, switches back to select view and reloads projects.
   - Show error message on failure

   **View: prompt** (x-show="$store.app.view === 'prompt'")
   - Project name heading with back button to select view
   - Phase suggestion: show "$store.app.phaseSuggestion.suggestion.phase_name" with status badge and reason. If null, show "No phase suggestion available".
   - Context preview: "Show context" toggle button. When toggled, show a details/summary or collapsible section displaying contextData (workspace, claude_md truncated, git_log). Load context lazily on first toggle via loadContext().
   - Prompt textarea (required, rows=5)
   - Mode select: autonomous/supervised (default autonomous)
   - "Run Task" submit button calls submitTask(prompt, mode)

   **View: running** (x-show="$store.app.view === 'running'")
   - "Task Running" heading with task status badge
   - Cancel button (visible when status is running/queued)
   - Approval UI (visible when approvalPending is true): show approvalData in a pre block, with Approve/Reject/Continue buttons calling sendApproval(decision). Copy the exact approval pattern from task_detail.html.
   - Streaming log output: pre.log-output with code element showing logText. Auto-scroll on new content using $nextTick and scrollTop.
   - "Back to Projects" button (visible when task is completed/failed/cancelled). Does NOT cancel running tasks. Calls backToSelect().

4. **Alpine.store('app'):** Define in a script block before Alpine CDN loads (using defer, Alpine auto-inits). Use document.addEventListener('alpine:init', ...) pattern from RESEARCH.md.

   State properties: view, selectedProject, projects, templates, phaseSuggestion, contextData, showContext, activeTaskId, logText, taskStatus, approvalPending, approvalData, ws, error

   Methods -- implement ALL of these following the patterns from RESEARCH.md code examples:
   - loadProjects() -- GET /projects, credentials: 'same-origin'
   - selectProject(project) -- set selectedProject, switch to prompt, call loadSuggestion()
   - loadTemplates() -- GET /templates, credentials: 'same-origin'
   - createProject(name, description, template) -- POST /projects, on success reload projects and switch to select
   - loadSuggestion() -- GET /projects/{id}/suggested-phase, credentials: 'same-origin'
   - loadContext() -- GET /projects/{id}/context, credentials: 'same-origin'
   - submitTask(prompt, mode) -- POST /tasks with project_id from selectedProject, on success set activeTaskId, clear logText, set taskStatus='running', switch to running view, call connectWS(task.id)
   - connectWS(taskId) -- Exact WS pattern from task_detail.html: sessionStorage token, ws:// protocol, onmessage handling chunk/status/approval_required, onclose with auto-reconnect when status is running/queued/awaiting_approval
   - sendApproval(decision) -- POST /tasks/{id}/approve
   - cancelTask() -- POST /tasks/{id}/cancel
   - backToSelect() -- close WS if open, reset activeTaskId/logText/taskStatus, switch to select, reload projects

5. **Helper functions** (outside Alpine.store, referenced in templates):
   - timeAgo(dateStr) -- relative time display (from RESEARCH.md)
   - stackBadges(stack) -- split comma-separated stack string into array

**Key implementation notes:**
- All fetch() calls MUST include `{ credentials: 'same-origin' }` for HTTP Basic Auth
- The WS token pattern uses sessionStorage with manual prompt fallback -- copy exactly from task_detail.html
- Auto-scroll the log container on new chunks using $nextTick and scrollTop = scrollHeight
- Use Pico CSS semantic elements: article for cards, hgroup for headings, details/summary for collapsible content
- The script tag with Alpine.store must use document.addEventListener('alpine:init', ...) and appear BEFORE the Alpine CDN script tag
