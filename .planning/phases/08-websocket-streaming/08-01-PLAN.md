<task type="auto" tdd="true">
  <name>Task 1: ConnectionManager and WebSocket endpoint with auth and heartbeat</name>
  <files>src/server/connection_manager.py, src/server/routers/ws.py, src/server/dependencies.py, src/server/app.py, tests/test_websocket.py</files>
  <behavior>
    - test_connection_manager_lifecycle: ConnectionManager.connect adds websocket to task set, disconnect removes it, empty sets are cleaned up
    - test_connection_manager_send_chunk: send_chunk delivers JSON {"type": "chunk", "data": text} to all connected sockets for a task_id
    - test_connection_manager_send_status: send_status delivers JSON {"type": "status", "data": status_str} to all connected sockets
    - test_connection_manager_prunes_dead: send_chunk removes sockets that raise on send
    - test_ws_connect_with_auth: WebSocket at /ws/tasks/{task_id}?token={base64_creds} accepts connection with valid credentials
    - test_ws_rejects_invalid_auth: WebSocket with invalid/missing token is rejected with close code 1008
    - test_disconnect_cleanup: After WebSocket disconnect, ConnectionManager no longer has that socket in its set
    - test_heartbeat_sends_ping: Connected WebSocket receives {"type": "ping"} message within heartbeat interval
  </behavior>
  <action>
    1. Create `src/server/connection_manager.py`:
       - Class `ConnectionManager` with `_connections: dict[int, set[WebSocket]]`
       - `async connect(task_id, websocket)` -- calls `await websocket.accept()`, adds to set
       - `disconnect(task_id, websocket)` -- sync, removes from set, deletes key if empty
       - `async send_chunk(task_id, text)` -- sends `{"type": "chunk", "data": text}` to all sockets, prunes dead ones
       - `async send_status(task_id, status)` -- sends `{"type": "status", "data": status}` to all sockets, prunes dead ones
       - `has_connections(task_id) -> bool`

    2. Add `verify_ws_token` dependency to `src/server/dependencies.py`:
       - Takes `websocket: WebSocket` and `token: str = Query(...)`
       - Decodes base64 token as "username:password"
       - Validates with `secrets.compare_digest` against settings (same creds as HTTP Basic Auth)
       - Raises `WebSocketException(code=status.WS_1008_POLICY_VIOLATION)` on failure
       - Returns username on success

    3. Create `src/server/routers/ws.py`:
       - `ws_router = APIRouter()`
       - `@ws_router.websocket("/ws/tasks/{task_id}")` with `Depends(verify_ws_token)`
       - Gets ConnectionManager from `websocket.app.state.connection_manager`
       - After connect: check task status via TaskRepository -- if completed/failed/cancelled, send final status message and close
       - Spawns heartbeat task: `asyncio.create_task(_heartbeat(websocket, interval=25.0))` that sends `{"type": "ping"}` every 25s
       - Main loop: `await websocket.receive_text()` (keeps connection open; messages ignored in Phase 8)
       - `except WebSocketDisconnect` + finally: cancel heartbeat, call `manager.disconnect(task_id, websocket)`

    4. Update `src/server/app.py`:
       - Import `ws_router` from `src.server.routers.ws`
       - In lifespan: create `ConnectionManager()` and store as `app.state.connection_manager`
       - In `create_app()`: `app.include_router(ws_router)`

    5. Write tests in `tests/test_websocket.py`:
       - Use Starlette `TestClient` for WebSocket tests (NOT httpx AsyncClient -- it does not support WebSocket)
       - Use `app_with_pool` fixture pattern from test_task_endpoints.py (with sync wrapper since TestClient is sync)
       - For ConnectionManager unit tests: use `AsyncMock` for WebSocket objects
       - For heartbeat test: use short interval (0.1s) and check received message within timeout
       - Auth token: `base64.b64encode(b"admin:changeme").decode()`
  </action>
