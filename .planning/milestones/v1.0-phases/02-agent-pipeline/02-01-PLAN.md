# Plan 02-01: Agent Config & Base Class

## Goal
Create a configuration-driven agent system where agents are defined declaratively (AGNT-05) and a base class handles the common invoke-parse-persist lifecycle.

## Tasks

1. Create `src/agents/config.py`:
   - AgentConfig dataclass: name, system_prompt, output_sections (list[str]), next_agent (optional)
   - AGENT_REGISTRY: dict[str, AgentConfig] with PLAN, EXECUTE, REVIEW pre-registered
   - `get_agent_config(name: str) -> AgentConfig`

2. Create `src/agents/base.py`:
   - BaseAgent class with:
     - `__init__(config: AgentConfig, db: aiosqlite.Connection, project_path: str)`
     - `async run(prompt: str, session_id: int) -> AgentResult`
     - AgentResult dataclass: agent_name, raw_output, sections (dict), handoff (str|None)
   - run() flow: build system prompt with context -> invoke_claude_with_retry -> extract_sections -> persist to DB -> return AgentResult

3. Create `tests/test_agent_config.py`:
   - Test registry contains all 3 agents
   - Test get_agent_config raises for unknown agent
   - Test AgentConfig fields

4. Create `tests/test_base_agent.py`:
   - Test run() with mocked Claude CLI produces AgentResult with correct sections
   - Test run() persists output to DB
   - Test handoff extraction from output sections

## Files to Create
- src/agents/__init__.py
- src/agents/config.py
- src/agents/base.py
- tests/test_agent_config.py
- tests/test_base_agent.py

## Success Criteria
- Adding a new agent requires only adding an entry to AGENT_REGISTRY
- BaseAgent.run() returns structured AgentResult with parsed sections
- All new tests pass
